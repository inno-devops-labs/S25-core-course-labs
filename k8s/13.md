- explanation of how ArgoCD handles configuration drift vs. runtime events:

  Argo CD handles **configuration drift** (manual changes diverging from Git) by detecting and correcting it (if auto-sync is enabled). For **runtime events** (e.g., pod restarts, node failures), Argo CD does not interveneâ€”Kubernetes manages these automatically. Configuration drift affects the desired state; runtime events do not.
